Проект веб-сайта "Интерактивная карта кладбища"
Описание проекта
Проект представляет собой веб-сайт с интерактивной картой кладбища, позволяющий пользователям искать и получать информацию о захоронениях. Система будет полностью на русском языке и предоставит функционал для работы как обычным пользователям, так и администраторам. Также доступен гостевой режим с ограниченным функционалом.
Основной функционал
Карта и отображение захоронений
1.	Интерактивная карта на основе Яндекс Карт (API ключ: 69436f8e-f6b6-41bf-9606-13f8a73cc804)
2.	Стартовое положение карты - Читинское центральное кладбище с координатами: 52.033635, 113.501049
3.	Захоронения отображаются в виде полигонов разной формы (могилы), а не точек
4.	Основная часть экрана занята картой, информация отображается в сайдбаре
5.	При нажатии на полигон (могилу) информация о захоронении отображается в сайдбаре
Информация о захоронениях
1.	ФИО погребенного
2.	Даты рождения и смерти
3.	Общее описание (добавляемое администратором)
4.	Личные заметки пользователя (видимые только ему)
Система пользователей
1.	Гостевой режим
•	Просмотр карты и информации о захоронениях
•	Поиск захоронений по ФИО и датам
•	Без возможности добавления заметок, избранного и предложения редактур
2.	Регистрация и авторизация
•	Регистрация по email с подтверждением
•	Авторизация через email/пароль
•	Восстановление пароля
•	Настройка профиля пользователя
3.	Роли пользователей
•	Гость (без регистрации)
•	Зарегистрированный пользователь
•	Администратор
Пользовательский функционал (для зарегистрированных пользователей)
1.	Добавление личных заметок к захоронениям
2.	Добавление захоронений в избранное
•	Выделение избранных захоронений на карте особым цветом/стилем
•	Быстрый доступ к списку избранных
3.	Возможность предлагать редактуру описания существующих захоронений через специальную форму
4.	Получение уведомлений о статусе предложенных редактур
Функционал администратора
1.	Рисование и добавление новых полигонов (могил) на карту
2.	Редактирование и удаление полигонов
3.	Добавление и редактирование информации о захоронениях (видимой всем пользователям)
4.	Модерация предложенных пользователями редактур описаний захоронений
•	Одобрение с обновлением информации о захоронении
•	Отклонение с отправкой уведомления пользователю
5.	Управление пользователями (блокировка, изменение ролей и т.д.)
Система поиска
1.	Поиск по ФИО
2.	Поиск по датам рождения/смерти
3.	Фильтрация по избранным захоронениям (для авторизованных пользователей)
Технологическая архитектура
Бэкенд
•	Язык программирования: Python
•	Фреймворк: Django + Django REST Framework
•	База данных: SQLite3
•	API: RESTful API для взаимодействия фронтенда с бэкендом
•	Аутентификация: JWT токены для API + сессионная аутентификация для веб-интерфейса
Фронтенд
•	HTML/CSS/JS
•	Библиотеки:
•	Яндекс Карты API
•	JavaScript-фреймворк (например, Vue.js или React)
•	Bootstrap или другой CSS-фреймворк для адаптивного дизайна
Структура базы данных
Модели данных
1.	Пользователь (расширение стандартной модели Django User)
•	ID
•	Email
•	Пароль (хэшированный)
•	Имя пользователя
•	Роль (Администратор/Пользователь)
•	Статус активации (активный/требует подтверждения)
•	Дата регистрации
•	Последняя активность
2.	Захоронение
•	ID
•	Координаты полигона (JSON-массив точек для построения полигона)
•	ФИО погребенного
•	Дата рождения
•	Дата смерти
•	Общее описание (доступно всем)
•	Дата добавления
•	Добавлено администратором (ID)
3.	Личные заметки
•	ID
•	ID захоронения (внешний ключ)
•	ID пользователя (внешний ключ)
•	Текст заметки
•	Дата создания/обновления
4.	Избранные захоронения
•	ID
•	ID захоронения (внешний ключ)
•	ID пользователя (внешний ключ)
•	Дата добавления в избранное
5.	Предложенные редактуры описаний
•	ID
•	ID пользователя (внешний ключ)
•	ID захоронения (внешний ключ)
•	Предлагаемое описание
•	Статус (на модерации, одобрено, отклонено)
•	Причина отклонения (если применимо)
•	Дата предложения
6.	Уведомления
•	ID
•	ID пользователя (внешний ключ)
•	Тип уведомления
•	Сообщение
•	Прочитано (да/нет)
•	Дата создания
•	Связанная сущность (ID захоронения или редактуры)
7.	Токены подтверждения регистрации
•	ID
•	Токен (уникальный)
•	ID пользователя (внешний ключ)
•	Дата создания
•	Дата истечения
